#### Домашнее задание
Использование Hibernate
#### Цель:
На практике освоить основы Hibernate.<br>Понять как аннотации-hibernate влияют на формирование sql-запросов.

#### Описание:
Создайте в классе Client поля:
* id
* name
* address
```java
class Address {
    private String street;
}
```
* phone
```java
class Phone {
    private String number;
}
```
Разметьте классы таким образом, чтобы при сохранении/чтении объекта Client<br>каскадно сохранялись/читались вложенные объекты.
#### ВАЖНО:
1. _Hibernate должен создать только три таблицы: для телефонов, адресов и клиентов._
2. _При сохранении нового объекта не должно быть update-ов.<br>Посмотрите в логи и проверьте, что эти два требования выполняются._
#### Дополнительно реализовано:
[Используя явную инициализацию ленивых коллекций с помощью `Hibernate.initialize`](https://github.com/av-starodub/2024-02-otus-java-starodubenko/blob/57cd6b032ba8bf2bf25382379e9a16e83e771461/HW13-JPQL/homework-template/src/main/java/ru/otus/crm/service/DbServiceClientImpl.java#L60) <br>
реализован контроль загрузки связанных данных. Таким образом решена проблема N+1 запросов.<br>
Этот способ выбран, потому что не требует дополнительных настроек Hibernate и может быть использован без Spring.
